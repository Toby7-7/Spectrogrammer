# 改动说明

本文记录当前仓库相对上游 `Spectrogrammer` 的主要改动，重点覆盖 Android 构建、运行时行为、中文 UI、频谱显示和配置系统。这里描述的是“当前分支已经做成了什么”，不是临时提交草稿。

## 项目定位变化
- 开发重心从通用实验性频谱查看器，转到 Android 手机竖屏使用场景。
- UI 语言改为中文优先，交互参考 Spectroid 的常用工作流。
- 处理链调整以“稳定显示真实频谱”为先，避免为了试验性优化牺牲基础可靠性。

## 1. 构建与仓库整理
- `Makefile`
  - 重做 Android 构建和打包流程，补齐 `init-submodules`、`doctor-android`、`push`、`run`、`logcat`、`clean`
  - 自动探测 Android SDK、NDK 和 build-tools
  - 编译 `src/java` 里的前台服务并打入 APK
  - 默认 APK 名仍为 `Spectrogrammer.apk`，应用标签改为 `频谱仪`
- `.gitmodules`
  - 子模块地址切到 HTTPS，拉取更直接
- `README.md`
  - 改成面向当前分支的中文说明，不再沿用上游旧文案
- `fastlane/metadata/...`
  - 仓库内发布说明与截图入口同步到当前版本定位

## 2. Android 运行时与系统行为
- `src/main_android.cpp`
  - 加入录音权限检查与启动门控
  - 接入系统字体，改善中文显示
  - 支持 Android 返回键：设置页返回主界面，主界面返回桌面
  - 增加回桌面保活、恢复前台后的状态恢复、亮屏控制
- `src/AndroidManifest.xml`
- `src/AndroidManifest.xml.template`
  - 增加 `RECORD_AUDIO`、`FOREGROUND_SERVICE`、`FOREGROUND_SERVICE_MICROPHONE`、`WAKE_LOCK` 等权限
  - 注册前台服务
- `src/java/org/nanoorg/Spectrogrammer/CaptureForegroundService.java`
  - 新增麦克风前台服务，配合后台采集使用

## 3. 配置持久化与默认值迁移
- `src/app/AppConfig.h`
- `src/app/AppConfig.cpp`
  - 新增版本化配置文件
  - 为旧配置添加迁移逻辑，修正早期默认值问题
  - 统一默认值：
    - 音频源 `默认`
    - 采样率 `自动 / 48000 Hz`
    - FFT `4096`
    - 平滑因子 `0.10`
    - 峰值标记来源 `短时峰值`
    - 峰值保持开启，默认 `4 秒` 回落
    - 后台采集开启
  - 对配置范围做兜底：
    - `滚动速度` 下限 `2 ms`
    - `FFT` 上限 `8192`
    - `峰值回落时长` 范围 `0` 到 `120 s`
    - `0` 表示 `不回落`

- `src/app/Processor.h`

- `src/app/fft.h`

  - 目标是优先保证频谱结果可信，再谈额外优化

## 5. 主界面与交互重做
- `src/app/Spectrogrammer.cpp`
- `src/app/Spectrogrammer.h`
  - 主界面改为中文
  - 顶部工具栏精简为四个按钮：`暂停/继续`、`清除峰值`、`音频`、`显示`
  - 增加状态行，显示输入采样率、当前视图范围、FFT 大小和频率分辨率
  - 音频和显示设置改为独立页面，并补上返回主界面流程
  - 设置页增大字体、控件和边距，适配手机触控
  - 去掉此前不需要的历史浏览功能和 `全频` 按钮
- 主图交互
  - 单指点击或拖动，可移动手动游标
  - 游标会在频谱图和瀑布图上同时显示纵线，并显示当前频率与 dB
  - 双指左右可缩放和平移频率视图，便于观察局部频段
- 峰值相关
  - 增加峰值保持曲线
  - 峰值标记可在 `实时峰值` 和 `短时峰值` 间切换
  - 峰值回落时长支持可调，也支持 `不回落`

  - 切后台再返回后可恢复原有瀑布图与频谱显示

## 7. 当前可调功能总览
- 音频源：`默认 / 通用 / 语音识别 / 摄像机 / 未处理`
- 采样率：`自动` 或固定档位，最高可选到 `384000 Hz`，但实际是否可用取决于设备
- FFT 大小：`128` 到 `8192`
- 抽取级数：`0` 到 `5`
- 窗函数：`矩形窗 / 汉宁窗 / 汉明窗 / 布莱克曼-哈里斯窗`
- 滚动速度：`2 ms` 到 `250 ms`
- 指数平滑因子：`0.00` 到 `0.95`
- 频率轴刻度：`线性 / 普通对数 / 音乐对数 / Mel / Bark / ERB`
- 瀑布图高度：`1/3 屏 / 1/2 屏 / 2/3 屏`
- 峰值标记数量：`0 / 1 / 3 / 5`
- 峰值标记来源：`实时峰值 / 短时峰值`
- 峰值回落时长：`0` 到 `120 s`
- 运行控制：`显示瀑布图 / 后台采集 / 保持亮屏`

## 8. 当前状态与注意事项
- 当前仓库以 Android 手机使用体验为优先目标，Linux 构建入口仍保留。
- 高采样率档位和 `未处理` 音频源是否可用，仍依赖具体设备驱动。
- 这份说明聚焦源码和功能改动，不包含 `.codex`、临时 `dex`、APK 构建产物等工作区临时文件。
